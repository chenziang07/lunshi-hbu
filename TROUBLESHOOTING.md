# 故障排查指南

## 摄像头断连问题

### 问题描述
摄像头频繁断连，尤其是在使用 Ctrl+Z 中断程序后。

### 原因分析
1. **Ctrl+Z 挂起进程** - SIGTSTP 信号会挂起进程而不是终止它，导致摄像头资源未释放
2. **资源泄漏** - 挂起的进程仍然占用摄像头设备，下次运行时无法访问

### 解决方案

#### 1. 正确停止程序
**推荐方式：使用 Ctrl+C**
```bash
# 运行程序
python main.py

# 停止时按 Ctrl+C（不要用 Ctrl+Z）
```

Ctrl+C 会触发 SIGINT 信号，程序会正确清理资源。

#### 2. 如果已经按了 Ctrl+Z
```bash
# 查看挂起的任务
jobs

# 恢复并终止（假设是任务 1）
fg %1
# 然后按 Ctrl+C

# 或者直接杀死
kill %1
```

#### 3. 清理残留的摄像头资源
```bash
# 运行清理脚本
python cleanup_camera.py
```

#### 4. 检查是否有残留进程
```bash
# Linux
ps aux | grep python
kill -9 <PID>

# Windows
tasklist | findstr python
taskkill /F /PID <PID>
```

## 掉台/拿起后自转问题

### 问题描述
机器人掉下台面或被拿起后，电机继续转动。

### 原因分析
1. **传感器读数异常** - 离开台面后，灰度传感器可能返回全 0 或异常值
2. **缺少安全检查** - 之前的代码没有检测传感器异常状态

### 已实施的解决方案

#### 1. 传感器异常检测
程序现在会检测以下异常情况并自动停止：
- 所有传感器读数为 0
- 任何传感器读数超出正常范围 (0-1.1)
- 传感器读取失败（异常）

#### 2. 安全停止机制
- 检测到异常时立即停止电机
- 在 `patrol.py` 和 `push.py` 中都添加了保护

#### 3. 异常处理
- 所有传感器读取都包裹在 try-except 中
- 电机控制失败时会自动停止

### 测试建议
1. 正常运行时观察日志，确认传感器读数正常
2. 拿起机器人，应该看到 "WARN: abnormal sensor readings" 并停止
3. 放回台面，程序应该恢复正常运行

## 最佳实践

1. **启动程序**
   ```bash
   python main.py
   ```

2. **停止程序**
   - 使用 Ctrl+C（推荐）
   - 不要使用 Ctrl+Z

3. **调试模式**
   ```bash
   # 在 config.py 中设置
   DEBUG = True
   ```
   可以看到详细的传感器读数和状态信息

4. **遇到问题时**
   - 检查终端输出的错误信息
   - 运行 `cleanup_camera.py` 清理资源
   - 检查是否有残留的 Python 进程

## 常见错误信息

| 错误信息 | 含义 | 解决方法 |
|---------|------|---------|
| `WARN: camera failed to open` | 摄像头被占用或不存在 | 运行 cleanup_camera.py |
| `ERROR: read_gray failed` | 灰度传感器读取失败 | 检查传感器连接 |
| `WARN: abnormal sensor readings` | 传感器数据异常 | 检查机器人是否在台面上 |
| `ERROR: set_speed failed` | 电机控制失败 | 检查电机连接和驱动 |
